# Lab Project: Framework Builder Harness / 贯穿实验：框架构建 Harness

## Project Goal / 项目目标

**中文**：构建一个帮助维护 Agent Harness Atlas 框架仓库的 Agent Harness。它能够读取框架结构、生成章节草稿、检查双语一致性、运行构建、提出改进建议，并在高风险发布动作前请求人工审批。

**English**: Build an Agent Harness that helps maintain the Agent Harness Atlas framework repository. It can read framework structure, generate chapter drafts, check bilingual consistency, run builds, propose improvements, and request human approval before high-risk publishing actions.

## Progressive Build / 分阶段构建

1. Define task contract and boundaries. / 定义任务契约和边界。
2. Implement minimal loop. / 实现最小闭环。
3. Add context pipeline. / 加入上下文管线。
4. Add tool gateway and permission matrix. / 加入工具网关与权限矩阵。
5. Add state, session, memory, and artifacts. / 加入状态、会话、记忆与产物。
6. Add runtime guards and recovery. / 加入运行时保护与恢复。
7. Package writing and review skills. / 封装写作与审查技能。
8. Add deterministic workflow for build and publish. / 为构建与发布添加确定性工作流。
9. Split roles only where boundaries differ. / 仅在边界不同处拆分角色。
10. Add trace, eval, security, and production architecture. / 加入跟踪、评测、安全与生产架构。
