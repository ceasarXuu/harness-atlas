Model×Harness =Agent

模型只产生可能性。Harness 把可能性接入任务、工具、状态、权限、验证与反馈，变成可执行、可观测、可改进的智能体。

定义 Harness = Context + Tools + State + Policy + Eval + Loop
目标 Agent Work = Reasoning × Environment × Feedback
外部化

把任务状态、证据和决策从长上下文中抽离出来。

闭环化

让执行结果回流到规划、验证和修复。

工程化

用权限、日志、测试和评估约束智能行为。

适合读者：开发者、独立开发者、产品经理、工程团队。
