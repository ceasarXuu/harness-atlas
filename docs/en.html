Model×Harness=Agent

A model produces possibilities. A harness connects those possibilities to tasks, tools, state, policy, verification, and feedback, turning them into executable, observable, improvable agent work.

Definition Harness = Context + Tools + State + Policy + Eval + Loop
Target Agent Work = Reasoning × Environment × Feedback
