## Positioning

**Harness Atlas** does not focus on a single product or vendor. It studies the general engineering pattern behind AI agents.

It explains how context, tools, state, memory, planning, evaluation, human oversight, and governance can be combined into a practical, controllable, and measurable agent system.

Current AI Coding tools already show early forms of Harness: repository context management, terminal and file-system access, tool invocation, test execution, code review loops, memory, permission checks, and human confirmation.

Harness Atlas starts from these concrete scenarios and expands toward broader software engineering systems.

## What is Agent Harness

> Agent Harness is a structured engineering environment that organizes model intelligence into controlled, reusable, and verifiable task execution.

A typical Harness includes:

| Component | Role |
|---|---|
| Context Management | Selects, compresses, injects, and refreshes task-relevant information |
| Tool Orchestration | Invokes tools, handles failures, controls permissions, and recovers from errors |
| State Management | Tracks task, file, execution, session, and project state |
| Planning Loop | Decomposes goals, performs actions, observes results, and revises plans |
| Knowledge Integration | Connects project knowledge, historical decisions, team conventions, and references |
| Evaluation and Feedback | Validates outputs through tests, reviews, metrics, and human feedback |
| Human-in-the-loop | Enables approval, intervention, supervision, and collaborative decision-making |
| Governance | Manages security boundaries, auditability, cost, compliance, and operational risk |

## Why It Matters

Large language models provide reasoning and generation capabilities, but real engineering tasks also require:

- stable access to project context
- reliable tool execution
- long-running task state management
- traceable decision records
- reusable workflows and working patterns
- testing and validation
- permission and risk control
- human-agent collaboration
- integration with existing engineering toolchains and team conventions

**Harness converts model capability into engineering capability.**
