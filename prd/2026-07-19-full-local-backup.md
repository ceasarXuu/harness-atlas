# PRD：本机重置前完整开发现场备份

- 状态：Ready for implementation
- 创建：2026-07-19
- 更新：2026-07-19
- Owner / requester：xuzhang
- 来源需求：在本机重置前，将 Harness Atlas 项目及其本地开发现场完整备份到外置盘，并提供可验证的恢复说明。

## Requester Review Summary

- 关键决策：以“无需依赖 GitHub 也能恢复 Harness Atlas 项目文件，并恢复仅与本项目直接关联的代理工作流与历史现场”为验收目标。
- 重要例外：运行中的 Unix socket、临时锁等不可持久化对象不作为有效开发数据恢复；对应服务重启后自动重建。
- 必须确认：备份目标为外置 APFS 卷 `xu-512` 下的独立项目专项目录，不覆盖既有整机备份。
- 状态说明：用户已明确要求完整备份、无需考虑备份盘加密，并要求恢复手册与 100% 开发现场恢复能力。

## 1. 背景与目标

本机即将重置。GitHub 可以恢复已推送源码，但不能覆盖 Git reflog、未跟踪/忽略内容、依赖现场、Codex/Claude 会话、自动化、项目技能、编辑器状态和本机认证配置。备份必须覆盖这些本地状态，并通过校验和恢复演练证明可用。

## 2. 成功标准

- 外置盘存在一个独立、时间标记清晰的项目备份目录。
- 完整仓库副本包含 `.git`、忽略目录、依赖、构建产物和所有隐藏文件。
- 项目专属 Codex/Claude 自动化、技能与会话被保留。
- 备份含来源清单、环境版本、SHA-256 校验清单和中文恢复手册。
- 从备份副本进行临时恢复演练后，Git 完整性和构建通过；测试结果与原工作区一致，并单独标记用户明确排除的 Downloads 外部依赖失败。

## 3. 用户与使用场景

主要使用者是项目维护者。重置 macOS、重新安装开发工具后，维护者应能仅依靠外置盘恢复项目目录、代理工作流、历史记录和必要配置，并继续开发、测试、构建和推送。

## 4. 范围

### 包含

- `/Users/xuzhang/harness-atlas` 完整目录。
- 仅与 Harness Atlas 直接关联的 Codex 自动化、项目技能、Codex 会话和 Claude 项目会话。
- 当前运行时、Homebrew、全局 npm、编辑器扩展等环境清单。
- 校验清单、恢复手册和恢复演练结果。

### 不包含

- macOS 系统文件和与本项目无关的个人媒体。
- `Downloads` 及其中已丢失的历史导入源包；用户明确要求不纳入本项目备份。
- 全量 Codex/Claude 状态库、全局会话、钥匙串、SSH/GitHub/npm 凭据、编辑器缓存及其他项目的开发数据。
- 可由应用自动创建且没有持久价值的 Unix socket、进程锁和瞬时 IPC 节点。
- 对外置盘进行加密、分区或清理。

## 5. 核心恢复流程

1. 新系统安装基础运行时和 Codex/Claude/编辑器。
2. 按原相对路径恢复项目目录与项目专属代理现场。
3. 修正新用户名或新 checkout 路径导致的绝对路径引用。
4. 运行 Git 完整性、依赖、构建和项目自动化检查。
5. 使用者自行恢复机器级认证并验证 GitHub 推送权限。

## 6. 信息结构

备份目录按 `项目仓库`、`项目关联代理现场`、`环境清单`、`校验与恢复文档` 分区，项目关联用户目录副本保留相对于 `/Users/xuzhang` 的路径。

## 7. 规则与状态

- 新建备份目录，不复用、不删除、不覆盖既有备份。
- 复制完成后才生成最终 SHA-256 清单；清单本身另记摘要。
- 任何复制、校验或恢复演练失败都会使备份状态保持“未验收”。
- 只收集能通过项目路径、项目自动化名称或项目技能名称直接证明归属 Harness Atlas 的本地状态。

## 8. 边界与恢复

- 外置盘空间不足或变为只读时立即停止，不删除任何来源或已有备份。
- 若文件在复制期间变化，执行第二轮同步并记录仍在变化的文件。
- Unix socket 只记录路径，不复制；应用恢复后重建。
- 机器级认证与全局工具状态不属于本项目恢复集，恢复后需由使用者重新配置。

## 9. 术语

- “完整开发现场”：项目文件、Git 元数据、依赖/构建现场、项目专属代理会话与自动化、环境版本清单的组合。

## 10. 验收标准

- Given 外置盘已挂载，when 备份完成，then 独立目录存在且未覆盖原有文件。
- Given 项目副本，when 执行 `git fsck --full` 与 `git status`，then 仓库对象完整且工作树状态与来源一致。
- Given 临时恢复副本，when 执行项目测试和构建，then 构建成功，测试结果与原工作区逐项一致，且 Downloads 外部依赖失败被明确标记为范围外既有问题。
- Given 校验清单，when 重新计算 SHA-256，then 全部文件匹配。
- Given 恢复手册，when 维护者按步骤操作，then 能定位每类数据的来源路径、目标路径和验证命令。

## 11. 评审清单

- 备份根目录是否位于外置盘 `xu-512` 的既有重置备份目录中？
- 是否同时保存完整副本与数据库一致性快照？
- 是否明确记录无法通过文件复制保证的系统登录状态？

## 13. 开放问题与风险

- `framework/import-manifest.json` 指向已不存在的 Downloads 历史源包，因此对应测试在当前原工作区已经失败；按用户要求该外部目录不纳入恢复范围。
- 当前 Codex 任务在备份结束后产生的最后少量会话尾记录不属于备份时点快照。

## 14. 实施说明

备份必须保留权限、时间戳、符号链接和扩展属性；最终结果通过 SHA-256、Git 检查、当前已通过的测试子集以及临时目录构建共同验收。Downloads 外部依赖导致的既有单项测试失败必须显式记录，不得误报为归档问题。
