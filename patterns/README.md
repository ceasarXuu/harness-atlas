# 设计模式

本目录沉淀 Agent Harness 的可复用设计模式。

设计模式来自产品拆解、研究资料、实验结果和工程实践，是连接 `products/`、`research/` 与 `course/` 的中间层。

## 目录

| 子目录 | 内容 |
|---|---|
| `context/` | 上下文相关模式 |
| `tools/` | 工具调用与权限模式 |
| `execution/` | 执行循环与任务管理模式 |
| `multi-agent/` | 多 Agent 协作模式 |
| `evaluation/` | 评估与 Benchmark 模式 |
| `governance/` | 安全、权限与治理模式 |

## 设计模式模板

- 问题
- 适用场景
- 解决方案
- 结构图
- 优点
- 风险
- 相关产品
- 相关研究
